# Data Warehouse Project

## Introduction

Data is everywhere — but raw data alone doesn't tell a story. This project was built with one goal in mind: transforming messy, unstructured retail data into a clean, reliable foundation that businesses can actually use to make better decisions.

Starting from raw CSV files, this project walks through the entire journey of data — from ingestion and cleaning, all the way to a structured data warehouse ready for analysis. Every step is intentional, every layer has a purpose.

## 📌 Overview

This project demonstrates the end-to-end process of building a **Data Warehouse** for a retail/e-commerce business using **SQL** and a **Star Schema (Kimball)** approach.

The pipeline covers:
- **Extraction** — Loading raw data from CSV source files
- **Transformation** — Cleaning, standardizing, and modeling the data
- **Loading** — Structuring data into fact and dimension tables

The final output supports business analysis across three key areas:
- 📈 **Sales & Revenue** — Understanding how the business performs over time
- 👥 **Customer Behavior** — Who buys, when, and how often
- 🛍️ **Product Performance** — Which products drive value and which don't

## 🛠️ Project Requirements

| Component | Detail |
|---|---|
| **Data Source** | CSV flat files (raw retail data) |
| **Database** | SQL-based (relational database) |
| **Modeling Approach** | Star Schema / Kimball Methodology |
| **Schema Structure** | Fact tables + Dimension tables |
| **Focus Areas** | Sales analysis, customer behavior, product performance |

**What you'll find in this repo:**
- Source CSV files (raw data)
- SQL scripts for data transformation & loading
- Schema diagram (ERD)
- Documentation for each layer of the pipeline
